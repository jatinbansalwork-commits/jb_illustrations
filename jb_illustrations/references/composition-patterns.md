# Composition Patterns and Originality Rules

## Base Structure Types

Pick one structure — don't mix too many.

### Workflow

For: input → process → output, content production, AI workflows, automation chains.

How to draw: inputs on the left, JB or a weird machine in the middle, output on the right, orange arrows for main flow.

### System Slice

For: information sources, filters, databases, renderers, agent system slices.

How to draw: only 3–5 core modules; JB performs one key action among them.

### Before / After

For: chaos/order, manual/auto, scattered/gathered, anxious/stable.

How to draw: chaos left, stability right, orange arrow in the middle. Character states can be more exaggerated.

### Character State

For: user pain points, creator burnout, too many tools, information anxiety, stuck → running.

How to draw: 2–4 small states, one short label each.

### Concept Metaphor

For: content factory, information warehouse, mental black box, workflow machine, auto-daily-report.

How to draw: one big weird object or machine, few inputs, one output. Must be memorable.

### Method Layers

For: methodology frameworks, system tiers, capability stacks, content system layers.

How to draw: stacked boxes — not a formal pyramid; JB bricks or builds beside them.

### Map Route

For: idea to launch, user path, content handoff path, learning route.

How to draw: one curved path, few nodes; JB pulls the rope or walks the route.

### Mini-Comic Panels

For: failure to success, real process, honest friction, before/after usage.

How to draw: 2–4 small scenes, one action per panel.

## Original Metaphor Method

Invent fresh metaphors from the current article every time — never copy old images.

### Three Steps

1. Turn the abstract concept into a physical action: stuck, leaking, getting heavy, sorting, settling, fermenting, opening, folding, unpacking, looping back.
2. Turn the system structure into a low-tech object: broken machine, cardboard box, drawer, pipe, post box, weird dial, scale, well, ladder, odd workstation.
3. Put JB in the action: not standing aside — stuck in the machine, pulling the wrong wire, guarding a door, carrying, patching, weighing, holding a ladder, logging, stuffing something into a weird device.

### Bangalore Everyday Object Pool (Use When It Fits)

- Filter coffee set (steel tumbler + davara, decoction flask, coffee filter).
- Yellow-black autorickshaw with locality signboard.
- BMTC bus stop bench and route number board.
- Jugaad wire, KR Market weighing scale, post-office stamp pad, tapri counter.

Pick only 1–2 objects. Don't crowd the frame.

### Universal Object Pool

- Cardboard box, drawer, old machine, funnel, scale, post box, door, well, ladder, pipe, yarn ball, gate, turntable, black box, hole punch, press, clothesline, weird workstation.

### JB Action Pool

- Pull, carry, stuff, scoop, press, weigh, stitch, cut, twist, guard, push, catch, unpack, mark, recycle.

Actions must serve the core meaning — weird for a reason, not weird for its own sake.

## Anti-Copy Rules

Don't open or copy `assets/examples/` by default. Those images only calibrate style: line density, whitespace, color restraint, JB's tone.

Unless the user explicitly says "copy this composition / use this case as a base," don't reuse these known layouts:

- Conveyor with two breakpoints
- JB pulling a judgment lever inside a content machine
- JB as a funnel sorting traffic/trust/conversion
- JB slicing a "material fish"
- JB holding a handoff path
- JB pulling three information sources
- Three JBs with megaphone / bridge / door
- JB stamping a handoff toolbox
- JB holding a sign at a pitfall path

Even for similar themes, invent new metaphors. "Handoff path" doesn't have to be a road — JB could attach a content tail to a door handle. "One input, many uses" doesn't need a fish — JB could press one paper ball into several shapes.
